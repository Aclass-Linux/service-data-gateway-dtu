## 3. 技术选型与理由

| 模块           | 技术方案                  | 选择理由                                                                 |
| -------------- | ------------------------- | ------------------------------------------------------------------------ |
| **表格组件**   | RevoGrid (MIT) + 虚拟滚动 | 框架无关，纯 JS；支持百万级数据流畅编辑；内置筛选、排序、单元格编辑。     |
| **图表组件**   | ECharts (Apache 2.0)      | 文档完善，社区活跃，支持大数据降采样，动态更新性能优异。                 |
| **本地存储**   | IndexedDB + Dexie.js      | 容量大（GB 级），异步不阻塞 UI；Dexie 简化复杂 API，提高开发效率。       |
| **实时通信**   | 原生 WebSocket            | 双向低延迟，浏览器原生支持；4KB/s 的数据量轻松应对。                     |
| **导入导出**   | 原生 JSON 操作 (无第三方库) | 导出为 `.json` 文件可直接使用 `JSON.stringify`，导入使用 `JSON.parse`，无需额外依赖，轻量且可控。 |
| **辅助工具**   | Lodash (debounce)         | 用于表格编辑防抖和图表批量更新定时器，优化性能。                         |

### 4.4 导入导出模块（改用 JSON）

#### 导出 JSON

- **触发方式**：点击顶部工具栏“导出 JSON”按钮。
- **数据来源**：读取当前表格组件的 `source` 数组（完整数据）。
- **生成文件**：
  - 将 `source` 通过 `JSON.stringify(data, null, 2)` 格式化为缩进 2 空格的 JSON 字符串。
  - 创建 Blob 对象，MIME 类型 `application/json`。
  - 动态生成 `<a>` 标签触发下载，文件名格式 `data_YYYYMMDD_HHmmss.json`。
- **特点**：完全前端完成，不依赖任何第三方库，体积小（10 万行数据约 5-10MB JSON 文本）。

#### 导入 JSON

- **触发方式**：点击“导入 JSON”按钮 → 弹出文件选择框，仅允许 `.json` 文件。
- **解析与校验**：
  - 使用 `FileReader` 读取文件内容为文本。
  - 使用 `JSON.parse` 解析为 JavaScript 数组。
  - 校验是否为数组，且长度不超过设定的最大行数（例如 20 万行），否则提示错误。
- **更新表格**：
  - 将解析后的数组直接设置为表格组件的新数据源。
  - 同时调用 `saveTableData` 将新数据保存到 IndexedDB。
- **提示**：导入前弹出确认框：“当前数据将被覆盖，是否继续？”

#### 可选：保留 Excel 导入能力（但不要求导出 Excel）

如果项目仍希望兼容用户上传的旧 Excel 文件，可以额外增加“导入 Excel”按钮（使用 SheetJS），但导出仅提供 JSON。此点为可选，不影响核心方案。

## 7. 异常处理与降级方案

| 异常场景                         | 处理措施                                                                 |
| -------------------------------- | ------------------------------------------------------------------------ |
| 导入的 JSON 文件格式错误         | 捕获 `JSON.parse` 异常，提示“文件格式错误，请检查是否为合法 JSON 数组”。 |
| 导入的 JSON 不是数组或为空       | 提示“数据无效，必须为非空数组”。                                         |
| 导入的数据行数超过限制           | 提示“数据量过大（超过 20 万行），请分批导入”。                           |
| IndexedDB 不可用（隐私模式）     | 降级为内存存储 + 提醒用户手动导出备份。                                  |
| WebSocket 连接失败               | 自动重连 3 次，若仍失败则停止并提示用户检查服务器地址。                  |
| 表格数据超过 10 万行导致卡顿     | 限制单次加载最大行数，提供“加载更多”分页按钮。                           |

## 11. 总结

本方案完全基于纯前端技术，**导出功能采用 JSON 格式**，无需引入 SheetJS 等额外库，使项目更轻量。导入/导出 JSON 文件具备简单、可靠、易调试的优点，同时与 IndexedDB 的数据格式天然兼容（均为 JavaScript 数组对象）。其他性能与架构优势保持不变。

**关键改动**：
- 导出按钮从 Excel 改为 JSON。
- 导入按钮默认支持 JSON，可选保留 Excel 导入（但非必需）。
- 移除对 SheetJS 的依赖，降低项目复杂度和体积。
